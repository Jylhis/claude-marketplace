---
description: "Explain code in plain English. Use when the user asks to explain, describe, or break down a piece of code."
---

When asked to explain code:

1. Start with a one-sentence summary of what the code does
2. Walk through the logic step by step
3. Highlight any non-obvious patterns or techniques
4. Note potential gotchas or edge cases

Use simple language. Avoid jargon unless the user is clearly technical.
